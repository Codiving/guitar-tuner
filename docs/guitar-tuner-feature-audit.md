# Guitar Tuner 기본 기능 점검 문서

작성 목적:
- 현재 구현이 기타 튜너의 기본 사용성에 충분한지 확인한다.
- 빠진 핵심 기능을 정리하고, 수정 우선순위를 제안한다.

## 현재 구현된 것

- 마이크 입력을 받아 실시간 피치 검출을 수행한다.
- 현재 감지된 음 이름을 표시한다.
- 옥타브, 센트 오차, Hz 값을 보여준다.
- 가장 가까운 기타 현을 추정해 하이라이트한다.
- 조율 바늘과 `IN TUNE` 상태를 제공한다.
- 시작/중지 토글이 있다.

## 기본 기능에서 부족한 점

### 1. 현재 음 알파벳 표시는 있지만, 설명이 충분히 친절하지 않다

지금은 `C#`, `A`, `E4` 같은 식으로 음 이름이 나온다.  
다만 초보자 입장에서는 다음 정보가 같이 있어야 더 쉽게 이해된다.

- 지금 잡힌 음이 목표 현보다 높은지 낮은지
- 얼마나 더 돌려야 하는지
- 소리가 너무 작아서 못 잡는 건지, 실제로 다른 음을 내고 있는지

권장 수정:
- 음 이름 옆에 `높음 / 낮음` 문구 추가
- `+12 cents`, `-8 cents`만 보여주지 말고 화살표 또는 안내 문구 추가
- 무음 상태와 검출 실패 상태를 구분해서 표시

### 2. 목표 현 선택이 없다

현재는 자동으로 가장 가까운 현을 추정한다.  
이 방식은 편하지만, 사용자가 어떤 현을 튜닝 중인지 명확히 지정할 수는 없다.

권장 수정:
- 6개 현을 직접 선택할 수 있게 한다
- `자동 감지`와 `수동 선택`을 둘 다 지원한다
- 표준 튜닝 외에 `Drop D`, `Half step down` 같은 프리셋을 넣는다

### 3. 신호 상태가 너무 비어 보인다

현재 로직은 일정 범위 밖이면 `pitch = null`로 떨어진다.  
이 경우 화면에서는 단순히 대기 상태처럼 보여서, 사용자는 앱이 정상 동작 중인지 알기 어렵다.

권장 수정:
- `무음`
- `입력 약함`
- `피치 불안정`
- `권한 거부`
를 분리해서 안내한다

### 4. 조율 방향이 직관적이지 않다

현재 메터는 센트와 바늘로만 방향을 암시한다.  
실사용에서는 `올려야 함 / 내려야 함`이 즉시 보여야 한다.

권장 수정:
- 센트 값 아래에 `조금 올리세요` / `조금 낮추세요` 문구 추가
- 바늘 색상이나 아이콘으로 방향을 표현
- `±5 cents` 이내일 때만 `정확함` 상태를 강조

### 5. 피치 검출 안정화가 필요하다

현재는 순간값을 바로 화면에 반영한다.  
소리가 흔들리면 표시가 튈 가능성이 있다.

권장 수정:
- 최근 몇 프레임 평균값 사용
- 일정 시간 유지된 값만 확정 표시
- 음이 바뀔 때만 화면을 갱신하도록 완충 처리

### 6. 기준 주파수 설정이 없다

현재는 A4 = 440Hz 기준이다.  
이 값은 일반적이지만, 일부 사용자는 다른 기준을 원할 수 있다.

권장 수정:
- `Reference A4` 설정 추가
- 최소한 432 / 440 / 442Hz 선택 가능하게 한다

### 7. 샤프/플랫 표기 일관성이 부족하다

현재 음 이름 계산은 기본적으로 `C#`, `D#` 같은 샤프 표기를 사용한다.  
그런데 반음 내림 프리셋에서는 `Eb`, `Ab`, `Db` 같은 플랫 표기가 섞여 있다.

이 상태에서는 같은 음을 서로 다른 방식으로 보여줄 수 있어서, 사용자가 보기에는 약간 헷갈릴 수 있다.

권장 수정:
- 프리셋별 표기 규칙을 정한다
- 기본 표기는 샤프로 통일하거나, 플랫 중심 프리셋에서는 플랫으로 통일한다
- 음명 변환 유틸을 따로 두고 `note name` 표기를 한 곳에서 관리한다

## 수정 우선순위

### 1순위

- 현재 음 이름을 더 명확하게 읽히도록 정리한다
- 조율 방향 문구를 추가한다
- 무음/입력 약함/권한 오류를 구분한다

### 2순위

- 목표 현 선택 UI를 만든다
- 표준 튜닝 프리셋을 넣는다
- 피치 검출 안정화를 적용한다

### 3순위

- 기준 주파수 설정을 추가한다
- 시각적 완성도를 높인다
- 모바일 환경에서 사용성 점검을 한다

## 권장 화면 구성

- 상단: 앱 이름 + 현재 상태
- 중앙: 현재 음 이름
- 보조: 현재 현, 목표 현, 센트 오차, Hz
- 하단: 방향 안내, 메터, 시작/중지 버튼

예시:

- `A4`
- `6번 줄 E`
- `+12 cents, 조금 낮추세요`
- `110.2 Hz`

## 결론

현재 프로젝트는 “음 이름을 보여주는 튜너”의 최소 형태는 갖추고 있다.  
하지만 기본 튜너로서 완성도를 높이려면 다음이 반드시 보강되어야 한다.

- 현재 음이 무엇인지 더 직관적으로 보여주기
- 조율 방향을 바로 이해할 수 있게 하기
- 무음/오류/불안정 상태를 분리하기
- 목표 현과 튜닝 프리셋을 제공하기

## 실사용 가능성 평가

현재 상태는 **가벼운 개인용 테스트나 데모 용도에는 충분히 사용 가능**하다.  
다만 일반 사용자에게 바로 배포하는 수준으로 보기에는 아직 몇 가지가 부족하다.

실사용에 가까워지려면 아래 항목이 더 필요하다.

- 무음 상태에서도 앱이 안정적으로 “기다리는 중”인지 “입력이 약한지”를 구분해야 한다
- 튜너가 흔들릴 때 화면이 튀지 않도록 안정화가 더 필요하다
- 플랫/샤프 표기 규칙을 통일해야 한다
- 목표 현 선택이 직관적이어야 한다
- 모바일 화면에서 작은 글씨와 버튼이 손가락으로 누르기 충분해야 한다
- 마이크 권한 거부 후 재시도 흐름이 더 친절해야 한다
- 튜닝 결과를 한눈에 이해할 수 있는 문구가 더 필요하다

즉, 지금도 “돌아가는 튜너”로는 쓸 수 있지만,  
“실제로 친구가 켜서 바로 쓰기 좋은 튜너”로 만들려면 **신호 안정성, 표기 일관성, 조작 흐름**을 한 번 더 다듬는 게 좋다.

이 순서대로 보완하면 초보자도 바로 쓸 수 있는 튜너가 된다.
